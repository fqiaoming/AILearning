### 1.1 什么是CO-STAR

```
CO-STAR是新加坡政府AI加速中心（AI Singapore）提出的
提示词框架，包含6个要素：

C - Context（上下文）
O - Objective（目标）
S - Style（风格）
T - Tone（语气）
A - Audience（受众）
R - Response（响应格式）
```

#### 1.2 CO-STAR的特点

```
✅ 全面性：6个要素覆盖所有关键维度
✅ 系统性：逻辑严密，层层递进
✅ 实用性：经过大量实践验证
✅ 灵活性：可以根据需求调整

适用场景：
- 复杂的内容生成任务
- 需要精确控制输出风格和语气
- 面向特定受众的内容
- 企业级应用
```

### 2. CO-STAR六大要素详解

#### 2.1 C - Context（上下文）

**定义：**
提供任务的背景信息，帮助AI理解整体环境。

**包含内容：**
```
1. 场景背景
   - 这是什么项目/产品
   - 在什么情况下使用

2. 历史信息
   - 之前做过什么
   - 当前状态如何

3. 约束条件
   - 有什么限制
   - 需要遵守什么规范

4. 相关资源
   - 可以参考什么
   - 依赖哪些系统
```

**示例：**

```
✅ 好的Context：
上下文：
这是一个面向中小企业的SaaS客户管理系统。
目标用户是销售人员和市场人员，大多没有技术背景。
当前问题：用户反馈帮助文档太技术化，看不懂。
技术栈：Web应用，支持PC和移动端。
现有资源：产品功能列表、用户反馈记录。

❌ 不好的Context：
需要写文档（没有背景信息）
```

---

#### 2.2 O - Objective（目标）

**定义：**
明确要达成的具体目标和成功标准。

**写法要点：**
```
1. 目标要SMART：
   - Specific（具体）
   - Measurable（可衡量）
   - Achievable（可达成）
   - Relevant（相关）
   - Time-bound（有时限）

2. 包含成功标准
   - 如何判断完成得好
   - 关键指标是什么
```

**示例：**

```
✅ 好的Objective：
目标：
编写一篇"快速开始"帮助文档，让新用户能在5分钟内
完成第一次成功的客户录入操作。

成功标准：
- 新用户看完后能独立完成操作
- 客服咨询"如何开始"的问题减少50%
- 新用户首次成功率从60%提升到90%

❌ 不好的Objective：
写个帮助文档（目标模糊，无法衡量）
```

---

#### 2.3 S - Style（风格）

**定义：**
指定输出内容的写作风格和表达方式。

**常见风格类型：**

```
1. 正式商务风格
   - 用于：商务报告、正式文档
   - 特点：客观、严谨、专业

2. 轻松口语风格
   - 用于：博客、社交媒体
   - 特点：亲切、易懂、有趣

3. 技术专业风格
   - 用于：技术文档、API文档
   - 特点：精确、简洁、术语准确

4. 教育引导风格
   - 用于：教程、培训材料
   - 特点：循序渐进、鼓励性、多举例

5. 新闻报道风格
   - 用于：新闻稿、报道
   - 特点：客观、简洁、信息密度高
```

**示例：**

```
✅ 好的Style设置：
风格要求：
- 写作风格：教程式，循序渐进
- 语言特点：通俗易懂，多用比喻和例子
- 表达方式：
  * 用"我们"而不是"系统"（拉近距离）
  * 用"点击"而不是"单击"（口语化）
  * 每个步骤都配截图说明（可视化）
- 参考风格：类似于「得到APP」的知识服务语言风格

❌ 不好的Style：
要轻松一点（太模糊）
```

---

#### 2.4 T - Tone（语气）

**定义：**
指定内容的情感基调和态度。

**常见语气类型：**

```
1. 友好热情
   适合：客服、营销
   例：很高兴为您服务！😊

2. 专业严谨
   适合：技术文档、法律文件
   例：根据系统检测结果...

3. 鼓励支持
   适合：教育、辅导
   例：做得很好！继续加油！

4. 同理关怀
   适合：客诉处理、心理咨询
   例：我理解您的感受...

5. 客观中立
   适合：新闻、报告
   例：数据显示...

6. 幽默风趣
   适合：社交媒体、创意内容
   例：这个bug也太调皮了吧！
```

**Style vs Tone的区别：**

```
Style（风格）= 怎么写（结构、措辞）
Tone（语气）= 什么态度（情感、立场）

例子：
Style：技术文档风格
Tone：友好耐心的语气

结果：
用清晰的技术语言（Style），但态度友好（Tone），
而不是冷冰冰的说明书。
```

**示例：**

```
✅ 好的Tone设置：
语气要求：
- 情感基调：温暖、耐心、鼓励
- 态度立场：站在用户角度思考
- 具体体现：
  * 遇到错误时，先安抚情绪再解决问题
  * 用"我们一起来..."而不是"你应该..."
  * 肯定用户的每一步尝试
  * 即使是拒绝，也要说明原因并提供替代方案

❌ 不好的Tone：
要友好（太抽象）
```

---

#### 2.5 A - Audience（受众）

**定义：**
明确目标受众是谁，他们有什么特点。

**受众分析维度：**

```
1. 技术水平
   - 技术专家
   - 一般用户
   - 技术小白

2. 领域知识
   - 行业专家
   - 有基础了解
   - 完全陌生

3. 年龄层次
   - 年轻人（18-30）
   - 中年人（30-50）
   - 老年人（50+）

4. 使用场景
   - 工作时查阅
   - 学习中使用
   - 紧急情况下求助

5. 心理状态
   - 有耐心慢慢学
   - 着急快速解决
   - 抵触情绪重
```

**示例：**

```
✅ 好的Audience设置：
目标受众：
- 身份：中小企业的销售人员
- 年龄：25-45岁
- 技术水平：会用微信、Excel，但不懂技术概念
- 使用场景：工作中遇到问题，希望快速找到答案
- 心理状态：时间紧迫，没耐心看长文
- 痛点：怕操作错误、怕丢失数据、怕太复杂学不会

针对性策略：
- 避免技术术语，用生活化比喻
- 步骤要短、要明确，每步不超过一句话
- 多用截图，少用文字
- 强调"不会有风险"、"可以撤销"，消除顾虑

❌ 不好的Audience：
面向用户（没有具体分析）
```

---

#### 2.6 R - Response（响应格式）

**定义：**
指定输出的具体格式和结构。

**格式类型：**

```
1. 纯文本格式
   - 段落式
   - 列表式
   - 问答式

2. 结构化格式
   - JSON
   - YAML
   - XML
   - Markdown
   - HTML

3. 代码格式
   - 完整程序
   - 代码片段
   - 配置文件

4. 表格格式
   - 对比表
   - 数据表
   - 时间表

5. 混合格式
   - 文字+代码
   - 文字+表格
   - 多级结构
```

**示例：**

```
✅ 好的Response设置：
响应格式：
结构：
1. 标题（简洁有力，不超过10个字）
2. 一句话概括（告诉用户这个功能是干什么的）
3. 快速开始（3步以内，每步一句话+截图）
4. 常见问题（2-3个，Q&A格式）
5. 进阶技巧（可选，点击展开）

格式规范：
- 使用Markdown
- 步骤用数字列表
- 重要提示用 ⚠️ 标注
- 技巧用 💡 标注
- 代码用```包裹
- 每个截图下方加说明文字

长度控制：
- 快速开始：200字以内
- 每个问题的回答：100字以内
- 总长度：500-800字

❌ 不好的Response：
用Markdown格式（没有具体要求）
```

### 3. CO-STAR完整示例

#### 示例1：产品帮助文档

```
【Context - 上下文】
产品：企业客户管理系统（CRM）
当前状况：新用户注册后，30%在第一次使用就放弃了
原因分析：不知道如何开始，界面功能太多感觉复杂
现有文档：有完整的功能说明书，但太详细，用户不愿意看
技术实现：Web应用，响应式设计，支持PC和移动端

【Objective - 目标】
目标：
编写一个"5分钟快速开始"教程，让新用户快速上手。

成功标准：
1. 新用户完成率从70%提升到90%
2. 首次成功录入客户的时间从15分钟降到5分钟
3. "如何开始"类型的客服咨询减少60%

可衡量指标：
- 教程完成率
- 首次成功时长
- 客服咨询量变化

【Style - 风格】
写作风格：
- 类型：手把手教程风格
- 特点：
  * 每个步骤都极其具体，不能有歧义
  * 多用"点击"、"填写"等动作词
  * 用"我们"称呼自己，拉近距离
  * 每个步骤都解释"为什么这样做"
- 参考：类似于iPhone的「使用手册」风格

【Tone - 语气】
语气：
- 基调：温暖、鼓励、支持
- 态度：站在用户角度，理解他们的顾虑
- 具体体现：
  * 开头先肯定用户（"恭喜你迈出第一步！"）
  * 每完成一步给予正面反馈
  * 预判用户可能的困惑，主动说明
  * 强调"很简单"、"只需要"，降低心理门槛

【Audience - 受众】
目标用户：
- 身份：中小企业销售人员、市场人员
- 年龄：25-40岁
- 技术水平：
  * 会用微信、Excel、Word
  * 不懂技术概念（API、数据库等）
  * 对新软件有畏难情绪
- 使用场景：
  * 刚注册完，想试试系统
  * 时间有限，希望快速上手
  * 担心操作错误
- 心理特点：
  * 没有耐心看长文档
  * 希望看到立即可用的效果
  * 需要信心鼓励

【Response - 响应格式】
输出格式：

# 5分钟快速开始 🚀

> 一句话概括功能（不超过20字）

## 第一步：[动作]（不超过10字）
[具体操作说明，一句话]
💡 小提示：[为什么这样做/注意事项]

## 第二步：[动作]
[具体操作说明]
💡 小提示：[...]

## 第三步：[动作]
[具体操作说明]
🎉 恭喜！你已经成功完成第一次[XX]！

## 常见问题 ❓
**Q1：[问题]**
A：[回答，50字以内]

格式要求：
- 使用Markdown
- 步骤总数不超过3个
- 每步说明不超过2句话
- 总长度500-800字
- 适当使用emoji增强亲和力（但不过度）
```

### 4. 三大框架全面对比

#### 4.1 要素对比

| 要素类别 | RTCF | CRISPE | CO-STAR |
|---------|------|--------|---------|
| 角色定位 | Role | Capacity | - |
| 任务描述 | Task | Statement | Objective |
| 背景信息 | Context | Insight | Context |
| 输出格式 | Format | - | Response |
| 风格语气 | - | Personality | Style + Tone |
| 受众分析 | - | - | Audience |
| 示例 | - | Experiment | - |
| **要素总数** | **4个** | **5个** | **6个** |

#### 4.2 特点对比

```
RTCF：
✅ 简洁实用，快速上手
✅ 灵活性高，适应性强
✅ 适合探索和快速迭代
⚠️ 输出稳定性一般
⚠️ 对复杂场景支持不足

CRISPE：
✅ 强调示例，准确率高
✅ 输出稳定，适合生产
✅ 特别适合分类/标注任务
⚠️ 需要精心设计示例
⚠️ Token消耗较多

CO-STAR：
✅ 最全面，覆盖所有维度
✅ 特别适合复杂内容生成
✅ 受众和风格控制精确
⚠️ 学习成本较高
⚠️ 可能过度设计
```

---

#### 4.3 适用场景对比

**RTCF 适用场景：**
```
✅ 快速原型开发
✅ 探索性任务
✅ 一次性任务
✅ 对输出格式要求不严格
✅ 个人使用，不需要标准化

示例：
- 头脑风暴
- 快速获取信息
- 学习探索
```

**CRISPE 适用场景：**
```
✅ 需要固定格式输出
✅ 分类/标注任务
✅ 需要高准确率
✅ 生产环境应用
✅ 批量处理

示例：
- 情感分析
- 内容分类
- 数据标注
- 模板化生成
```

**CO-STAR 适用场景：**
```
✅ 复杂内容创作
✅ 面向特定受众
✅ 需要精确控制风格和语气
✅ 企业级应用
✅ 多角色协作场景

示例：
- 产品文档
- 营销文案
- 教育内容
- 客服话术
```

### 5. 框架组合使用策略

#### 策略1：RTCF起草 + CRISPE示例

```
适用：需要快速验证想法，然后提升准确率

步骤：
1. 用RTCF快速写一版
2. 测试效果
3. 如果不稳定，添加CRISPE的示例
4. 再次测试，调优

优势：
- 快速迭代
- 逐步优化
- 兼顾效率和质量
```

#### 策略2：CO-STAR框架 + CRISPE示例

```
适用：复杂的生产环境应用

做法：
【Context】（CO-STAR）
背景信息...

【Objective】（CO-STAR）
目标...

【Style & Tone】（CO-STAR）
风格和语气...

【Audience】（CO-STAR）
受众分析...

【Examples】（CRISPE）
示例1：...
示例2：...

【Response】（CO-STAR）
输出格式...

优势：
- 全面系统
- 输出稳定
- 适合企业级应用
```

#### 策略3：三框架要素自由组合

```
根据实际需求，选择不同框架的优势要素：

必选要素（来自RTCF）：
- Task/Objective：任务必须明确

高优先级（根据场景选择）：
- Examples（CRISPE）：需要准确时必须有
- Audience（CO-STAR）：面向特定人群时必须有
- Style/Tone（CO-STAR）：内容创作时很重要

可选要素：
- Context：复杂任务才需要详细背景
- Role：影响相对较小
```

"""
三大提示词框架综合应用演示
展示如何根据场景选择合适的框架
"""

from openai import OpenAI
import os
from dotenv import load_dotenv

load_dotenv()

client = OpenAI(
    base_url=os.getenv("LOCAL_LLM_BASE_URL"),
    api_key=os.getenv("LOCAL_LLM_API_KEY")
)


def get_response(prompt: str, temperature: float = 0.7) -> str:
    """获取AI回复"""
    response = client.chat.completions.create(
        model=os.getenv("LOCAL_LLM_MODEL"),
        messages=[{"role": "user", "content": prompt}],
        temperature=temperature,
        max_tokens=1200
    )
    return response.choices[0].message.content


def demo_rtcf_quick():
    """演示1：用RTCF快速探索"""
    print("\n" + "="*80)
    print("场景1：快速探索阶段 - 使用RTCF框架")
    print("="*80)
    print("需求：快速了解一个技术概念")
    
    prompt = """【Role】你是一位Python专家
【Task】用简单的语言解释什么是装饰器
【Context】读者有Python基础但不懂装饰器
【Format】3段，每段不超过50字"""
    
    print(f"\n提示词：\n{prompt}")
    print("\n" + "-"*80)
    response = get_response(prompt, temperature=0.7)
    print(f"AI回复：\n{response}")
    print("\n💡 特点：简洁快速，适合探索")


def demo_crispe_production():
    """演示2：用CRISPE实现生产级准确性"""
    print("\n" + "="*80)
    print("场景2：生产环境 - 使用CRISPE框架（带示例）")
    print("="*80)
    print("需求：评论情感分类，要求准确率>90%")
    
    prompt = """【Capacity】你是情感分析专家
【Insight】电商评论分析，用于商品评价系统
【Statement】判断评论情感：正面/负面/中性
【Personality】客观准确
【Experiment - 示例】
示例1：
评论："这个产品超级好用，强烈推荐！"
分类：正面

示例2：
评论："质量太差了，后悔购买"
分类：负面

示例3：
评论："还行吧，没什么特别的"
分类：中性

---
请分类以下评论：
"外观不错，但是用起来有点卡，总体还可以"""
    
    print(f"\n提示词：\n{prompt}")
    print("\n" + "-"*80)
    response = get_response(prompt, temperature=0.1)
    print(f"AI分类：\n{response}")
    print("\n💡 特点：示例加持，准确率高")


def demo_costar_comprehensive():
    """演示3：用CO-STAR处理复杂内容创作"""
    print("\n" + "="*80)
    print("场景3：复杂内容创作 - 使用CO-STAR框架")
    print("="*80)
    print("需求：为产品写新手引导文案")
    
    prompt = """【Context - 上下文】
产品：智能笔记应用
用户群：大学生和职场人士
问题：新用户不知道如何开始使用
目标：降低使用门槛，提升留存率

【Objective - 目标】
编写"60秒快速开始"引导文案，让新用户立即体验到产品价值
成功标准：新用户首日留存率从40%提升到70%

【Style - 风格】
轻松活泼的教程风格，类似于B站UP主的讲解风格

【Tone - 语气】
友好、鼓励、有趣，用"我们"和"你"拉近距离

【Audience - 受众】
- 18-35岁，追求效率
- 手机重度用户
- 没耐心看长教程
- 需要立即看到效果

【Response - 响应格式】
格式：
# 标题（有趣吸引人）
## 第1步（一句话）
💡 为什么：[解释]
## 第2步（一句话）
💡 为什么：[解释]
## 第3步（一句话）
🎉 完成提示

总长度：200字以内
语气：轻松有趣，用emoji"""
    
    print(f"\n提示词（展示部分）：\n{prompt[:300]}...")
    print("\n" + "-"*80)
    response = get_response(prompt, temperature=0.8)
    print(f"AI输出：\n{response}")
    print("\n💡 特点：全面控制，适合复杂创作")


def demo_hybrid():
    """演示4：组合使用 - CO-STAR + CRISPE示例"""
    print("\n" + "="*80)
    print("场景4：企业级应用 - 组合CO-STAR和CRISPE")
    print("="*80)
    print("需求：客服机器人标准回复")
    
    prompt = """【Context】
企业客服机器人，处理售后咨询
要求回复标准化、专业化

【Objective】
对用户咨询给出规范回复
要求：态度友好、信息准确、引导明确

【Audience】
已购买产品的用户，可能有不满情绪

【Tone】
耐心、同理心、解决问题导向

【Examples - 示例】
示例1：
用户："我的订单还没到，急用怎么办？"
回复："我理解您的着急心情！我帮您查询一下订单：[订单号XXX]
目前在配送中，预计明天上午送达。如果明天上午还未到，请随时联系我，
我们会优先处理！您可以在'我的订单'中查看实时物流信息。"

示例2：
用户："这个产品怎么用不了？"

1. 具体是哪个功能用不了？
2. 出现了什么提示或现象？
您也可以查看使用指南：[链接]，或联系技术支持：[电话]"

---
现在请回复：
用户："我想退货，但是找不到退货入口"
"""
    
    print(f"\n提示词（展示部分）：\n{prompt[:400]}...")
    print("\n" + "-"*80)
    response = get_response(prompt, temperature=0.5)
    print(f"AI回复：\n{response}")
    print("\n💡 特点：结合两个框架的优势，企业级应用")


def demo_framework_selection():
    """演示5：框架选择决策树"""
    print("\n" + "="*80)
    print("演示5：如何选择合适的框架")
    print("="*80)
    
    print("""
决策树：

问题1：是探索性任务还是生产任务？
├─ 探索性 → 用RTCF（快速灵活）

def main():
